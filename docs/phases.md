# Tsumory 開発フェーズ

`mymanifesto.md`のコンセプト3点に照らして、「つぶやく → AIが日記にする → 作り直す」のコアループが完結することを最優先に切り分ける。

## フェーズ1(MVP): コアループの最短実装

目的: コンセプト3点が成立することを検証する。

1. ログイン・ログアウト(Spring Security + BCrypt)。初期ユーザーはFlywayマイグレーションでINSERTする。
2. つぶやき投稿(フォーム、100文字程度、投稿時刻自動付与)
3. 当日のつぶやき一覧表示(時系列)
4. つぶやきの編集(投稿時刻は保ったまま本文を直せる)
5. つぶやきの削除
6. 日記生成(Claude API、当日のつぶやきを1本に再構成)
7. 日記表示(当日1本)
8. 日記の再生成(同じ日付の日記を上書き)

### MVPで意図的に落とすもの

- **ユーザー登録UI**: 初期ユーザーはマイグレーションで投入する
- **過去日記の閲覧・カレンダー**: まず「今日1本」が成立するかが先
- **通知・リマインド**: まず自然に使われるかを見る
- **任意日付での日記生成**: 「今日まとめる」だけで概念検証は足りる

## フェーズ2: 体験を厚くする拡張

- AIによるつぶやきのカテゴリ自動分類
- 過去の日記一覧
- 日記の検索
- 遡り生成(任意日付での日記生成)

## 切り分けの判断基準

- **MVPに入れるか?** → 無いとコアループが回らないか
- **フェーズ2に回すか?** → コアループは回るが、続ける動機や振り返り価値を高める機能か
